# Правила агента — Content Factory

Этот файл определяет поведение AI-агента при работе с контентом Telegram-канала.

---

## Контекст проекта

**Автор:** Игорь Шорец — сомелье, wine director, Москва.  
**Канал:** экспертный контент о вине: выборы, дегустации, мнения, инсайды индустрии.  
**Цель:** ежедневная публикация одного поста по расписанию.

---

## Структура файлов

```
Content-factory/
├── agents.md           # этот файл
├── content_plan.md     # список идей с датами и статусами
├── posts/              # черновики и готовые посты (YYYY-MM-DD.md)
└── published/          # посты после публикации
```

---

## Поведение агента при получении идеи

1. **Уточнить дату** — если не названа, предложить ближайший свободный слот из `content_plan.md`
2. **Оформить пост** и сохранить в `posts/YYYY-MM-DD.md`
3. **Обновить `content_plan.md`** — добавить строку с датой, темой и статусом
4. **Ничего не публиковать** — только подготовить файл

---

## Формат файла поста

```markdown
---
date: YYYY-MM-DD
status: draft | ready | published
tags: [тег1, тег2]
---

[текст поста]
```

**Статусы:**
- `draft` — сырая идея, требует доработки
- `ready` — готов к публикации
- `published` — опубликован, перемещён в `published/`

---

## Голос и стиль

- **Тон:** сдержанный, экспертный, без пафоса
- **Длина:** 500–900 символов — оптимум для Telegram
- **Один тезис на пост** — не размывать тему
- **Конкретика:** регион, производитель, vintage, цифры — если уместно
- **Эмодзи:** минимально, только по смыслу (не для украшения)
- **Хэштеги:** 2–4 в конце, только если релевантны

---

## Чего агент НЕ делает

- Не публикует без явной команды «публикуй» / «опубликуй»
- Не изменяет готовый пост без просьбы
- Не придумывает факты о вине, производителях или регионах
- Не добавляет хэштеги без запроса
- Не меняет голос и стиль без явного указания

## Команды агенту

| Команда | Действие |
|---|---|
| «Идея: ...» | Оформить пост, сохранить как draft |
| «Готов» / «Сделай ready» | Сменить статус на ready |
| «Что на очереди?» | Показать ближайшие ready-посты |
| «Покажи план» | Вывести content_plan.md |
| «Архивируй YYYY-MM-DD» | Переместить в published/ |
